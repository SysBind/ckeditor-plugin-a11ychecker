Introduction

Accessibility Checker is an innovative solution that lets you inspect accessibility level of content created in CKEditor and immediately solve any issues that are found. This sample uses Quail as its accessibility tests library, but you can integrate any other similar library to achieve comparable results. Refer to the General Information section below for more information about default keyboard shortcuts and some configuration options.

General Information

Configuration

Accessibility Checker exposes some configuration options explained below. You can specify the configuration either globally (in the ckeditor.js file or a custom configuration file loaded with the config.customConfig option), or in-page directly in the call creating a CKEditor instance.

If you need more information on how to change the editor configuration, refer to the Setting CKEditor Configuration article.

Default Navigation Hotkeys

Working with Accessibility Checker can be really efficient if you use keyboard shortcuts to navigate it. The hotkeys are available when the editor has focus.

The default navigation hotkeys are:

  • Ctrl+Alt+E – Enables and disables the Accessibility Checker.
  • Ctrl+E – Moves to the next issue.
  • Ctrl+Shift+E – Moves to the previous issue.

Changing Tests

This example uses the Quail accessibility tests library. You may change these tests using the config.a11ychecker_quailParams configuration option. You just need to provide the guideline property, as in the example below: 

config.a11ychecker_quailParams = {
	guideline: [ 'imgHasAlt', 'aMustNotHaveJavascriptHref', 'aAdjacentWithSameResourceShouldBeCombined' ]
};

Changing Default Hotkeys

Accessibility Checker keyboard shortcuts can be customized in your configuration, like in the example below: 

// Override default Accessibility Checker hotkeys.
config.a11ychecker_keystrokes = {
	// Ctrl+Shift+O
	open: CKEDITOR.CTRL + CKEDITOR.SHIFT + 79,
	// Ctrl+ESC
	listen: CKEDITOR.SHIFT + 27,
	// Ctrl+Shift+[
	prev: CKEDITOR.CTRL + CKEDITOR.SHIFT + 219,
	// Ctrl+Shift+]
	next: CKEDITOR.CTRL + CKEDITOR.SHIFT + 221,
	// ESC
	close: 27
};
